## サービス概要
どんなサービスなのかを３行で説明してください。
同じコミュニティ内で生まれた感謝の気持ちを送るお手伝いをします。相手のことを思った画像を生成してメッセージと共にプレゼントします。
ユーザーは、3感謝メッセージ送ると一つのアイテム（ニット系のアイテムまたはキャラクターなど）を獲得できます。
アイテムは集めることもでき、人にプレゼントすることもできます。

## このサービスへの思い・作りたい理由
このサービスの題材となるものに関してのエピソードがあれば詳しく教えてください。
このサービスを思いつくにあたって元となる思いがあれば詳しく教えてください。

RUNTEQコミュニティをはじめ、どのコミュニティで感謝の気持ちを伝えたい時があります。
感謝のメッセージを送ることはありますが、プラスアルファーで何かを送りたい時が多々あります。実際のプレゼントを贈るほどではないけど、もう少し何かを送りたい。
そんな時に、このアプリで感謝のメッセージと相手のことを思って生成した画像をプレゼントすることができます。
私は昔から知らない人に話しかけてすぐに仲良くなれます。そして、仲良くなったさまざまな立場の方々から、お世話になることが多かったです。
その度に、自分のできる範囲で感謝を伝えてきました。言葉だったり自分の活動の成果を報告したり。
それをプログラミングに置き換えた時、自分のアプリーケーションで感謝の気持ちと共にプレゼントを贈ることでした。
これならどんなに時間がかかっても、どんなにエラーに苦しめられてもこれだったら、途中に投げ出さずに作れると確信しました。
このサービスによって、コミュニティの人々が今まで以上にコミュニティでいろんな形の感謝を伝えやすくなって欲しいと強く思っています。

## サービスの利用イメージ
ユーザーがこのサービスをどのように利用できて、それによってどんな価値を得られるかを簡単に説明してください。

- ユーザーはサイトにアクセス

*ログインした場合*
- 感謝状を書いてAIで画像生成
感謝状に相手の名前と感謝メッセージを書きます。
自分でタグを作る（感謝の内容やその人自身を連想させるようなAIが具体的な画像を生成しやすいキーワード）。
タグを元にAIで画像生成し、感謝状と共に掲示板投稿できる。

感謝メッセージの例：
〇〇さんいつもありがとうございます！この間、Rails基礎のメーラーの部分がわからなくて、〇〇さんさんに丁寧に解説していただけて理解することができました！
本当にありがとうございます！
タグの例：バトミントン, しば犬, 薬、ピンク、イチゴ、お酒
(タグには、その人の好きなものや関連するモチーフを入力する。)


- 感謝状を3回送ると、一つのアイテムを獲得
アイテムは、編み物をテーマにしたもの。

- アイテムをユーザー間でプレゼントしあう！
このアイテムは獲得したら自身のプロフィールに保存されます。自分でコレクションにもできますし、人にプレゼントすることもできます。
獲得したアイテムは、ユーザーのマイページで見ることができます。そこで、別のユーザーにアイテムをプレゼントすることができます。

- Xシェア機能
  - 生成された画像をXシェア。
  感謝状投稿後にshowページにある、「Xでメッセージを送る」ボタンをクリックする。
  - 獲得したアイテムや、コレクションしているアイテムをXシェア。
  ユーザーのマイページでXシェアボタンをクリックする。


- 検索機能
感謝状一覧画面で、名前と感謝のメッセージとタグを検索することができる。


*ログインしない場合*
感謝状一覧の閲覧。
検索機能。

## サービスの差別化ポイント・推しポイント
似たようなサービスが存在する場合、そのサービスとの明確な差別化ポイントとその差別化ポイントのどこが優れているのか教えてください。
独自性の強いサービスの場合、このサービスの推しとなるポイントを教えてください。

感謝の気持ちを言葉だけでなく、AIで生成した画像をプレゼントすることを目的に置いている点。
通常AIの画像生成は、アイコンなどの自分のために画像を作ることが多いですが、このサービスは人のために作る点が推しポイントです。
感謝のメッセージを送ることによって、アイテムが獲得できる点。
感謝を編むをテーマに、獲得アイテムは編み物をテーマにしたものにする。
加えて、3メッセージを送らないとアイテムを獲得できないので、ゲーム感覚で強制的に3人または3回感謝状を送りたくなる。
取得したアイテムをプレゼントできることによって、更なる交流のきっかけになる。
